DevOps / AI Flow

Docker、CI/CD 与低代码 AI Flow 学习路线

这篇笔记记录我从 IT 支持向技术项目支持补齐能力的路线。目标不是把自己包装成 AI 工程师， 而是理解容器、自动化部署和大模型工具流如何服务真实业务问题。

为什么学这些

现场 IT 支持经常面对的是“系统不可用、设备异常、用户不知道如何描述问题”。Docker、CI/CD 和 AI Flow 能帮助我理解应用交付、自动化检查和知识库问答，为后续参与 AI POC、内部工具或项目支持打基础。

学习拆解

  • Docker：镜像、容器、端口映射、volume、基础网络和本地服务启动。
  • CI/CD：用 GitHub Actions 或 GitLab CI 做静态站点检查、构建和部署。
  • Dify / AI Flow：梳理输入、知识库、提示词、检索和输出，做一个 IT 支持问答原型。
  • 文档资产：把 SOP、Runbook、故障复盘整理成可检索、可交接的结构化资料。

和制造/物流现场的关系

现场系统稳定性要求高，任何自动化或 AI 工具都必须能被追踪、验证和回滚。实习中积累的终端、网络、 日志、工单和跨部门沟通经验，是理解业务问题和编写有效技术文档的入口。

下一步计划

  1. 为个人博客补充 Dockerfile，并在本地验证容器启动。
  2. 把常见排障记录整理成 Markdown 知识库。
  3. 用 Dify 搭建“IT 支持问答助手”原型，输入故障描述后返回排查路径。
